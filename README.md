# K歌选歌（SingList）

一款**完全离线、本地优先**的 Flutter 跨端 MVP，用于管理 K 歌歌曲库、标签、歌单与 KQueue 队列，并支持刷歌 / 随机生成、分享与批量导入。

## 功能概览

应用底部有四个主 Tab：

| Tab | 说明 |
|-----|------|
| **曲库** | 歌曲增删改查、搜索、按标签筛选、批量操作、歌曲详情 |
| **标签** | 预设与自定义标签，查看标签下歌曲 |
| **歌单** | 普通歌单与 KQueue 队列管理 |
| **生成** | 刷歌模式与随机生成 KQueue |

### 曲库

- 新增 / 编辑 / 删除歌曲（以「歌名 + 歌手」唯一）
- 歌名 / 歌手关键词搜索（大小写、空格不敏感）
- 标签 FilterChip 快速筛选
- 长按进入批量模式：加标签、加入歌单、删除
- 单击进入**歌曲详情**（查看信息、管理标签、编辑）
- 右上角**设置**入口
- 支持**统一导入流程**（粘贴文本或选择 txt / csv 文件）

### 标签

- 预设标签：**开嗓**、**收尾**、**合唱**（不可改名 / 删除）
- 新建、修改、删除自定义标签
- 点击标签查看该标签下所有歌曲
- 显示每个标签关联的歌曲数量

### 歌单 / 队列

- **普通歌单**：歌曲不重复，可批量删除
- **KQueue 队列**：允许重复、支持拖拽排序、移动到指定位置、清空
- 队列支持**文本导出与分享**（`#K歌歌单` 标题 + 每行 `歌名 - 歌手`）
- 歌单页支持导入 KQueue

### 生成器

**刷歌模式**

- 来源：全部歌曲 / 按标签 / 按普通歌单
- 可选开嗓暖场曲（从「开嗓」标签随机抽取）
- 逐首标记：特别想唱 / 想唱 / 不想唱
- 支持撤销上一首
- 完成后按顺序生成 KQueue：开嗓暖场 → 特别想唱 → 想唱

**随机生成**

- 选择来源与数量
- 可开启「歌曲不重复出现」
- 可选随机开嗓曲
- 一键生成 KQueue

### 设置与数据维护

从曲库右上角进入设置页：

- **导出备份**：全库 JSON 备份，通过系统分享保存
- **从备份恢复**：选择 json 文件追加导入歌曲
- **合并重复歌曲**：按歌名 + 歌手归组，保留一条并合并关联
- **导入记录**：最近 3 次导入摘要

## 导入说明

统一入口：`showImportFlow()`（曲库、歌单页均可触发）

1. 选择**粘贴文本**或**选择文件**（txt / csv）
2. 预览解析结果（成功条数、失败行）
3. 选择导入目标：
   - **曲库**：仅写入歌曲库
   - **普通歌单**：写入曲库并加入指定歌单
   - **KQueue 队列**：写入曲库并追加到队列（可新建）

### 支持的文本格式

每行一首，解析规则（`lib/service/import_parser.dart`）：

```
稻香 - 周杰伦
晴天/周杰伦
歌名,歌手
歌名 歌手
```

- 忽略空行与 `#` 开头行（如 `#K歌歌单`）
- 支持 `-`、`/`、逗号、空格等分隔符
- CSV 文件按列解析

### KQueue 文本格式

导出示例：

```text
#K歌歌单
稻香 - 周杰伦
晴天 - 周杰伦
```

导入时同样识别该格式，并保留队列顺序（含重复条目）。

```
├── ui/
│   ├── pages/          # 各主页面与详情页
│   └── widgets/        # 通用组件、导入流程、iOS 风格控件
├── app.dart            # 应用入口与底部导航
├── main.dart
└── theme.dart

test/                   # 单元测试（解析、规范化、生成规则、去重等）
```

## 数据模型

- **Song**：`title`, `artist`, `titleNorm`, `artistNorm`（歌名 + 歌手唯一；规范化：trim、压缩空格、转小写）
- **Tag**：`name` 全局唯一
- **SongTag**：多对多，`(songId, tagId)` 唯一
- **Playlist**：`type = normal | kQueue`
- **PlaylistSongs**：普通歌单条目，`(playlistId, songId)` 不重复
- **QueueItems**：KQueue 条目，允许重复，含 `position` 用于排序

## 快速开始

### 环境要求

- Flutter SDK >= 3.22.0
- Dart SDK >= 3.3.0

### 安装与运行

```bash
# 获取依赖
flutter pub get

# 若修改了 Drift 表结构，重新生成代码
dart run build_runner build --delete-conflicting-outputs

# 运行（选择已连接设备或模拟器）
flutter run
```

### 测试

```bash
flutter test
```

当前测试覆盖：文本规范化、导入解析、KQueue 文本导入导出、队列生成规则、重复歌曲合并等。

## 设计原则

- **离线优先**：不依赖网络与账号，数据存于本地 SQLite
- **间接导入**：通过粘贴 / 文件导入歌单文本，而非直连音乐平台 API
- **K 歌场景**：围绕「选歌 → 刷歌筛选 → 生成 KQueue → 上台顺序」设计
